# ScholarshipHub workflow
This agentic workflow was created for the Foundrez: Agent Architect course specifically as a finale project for Module 1. Agent Master. The workflow was initially created using the Foundry portal and subsequently converted into code using the steps outlined at [Convert a YAML workflow to Agent Framework code](https://learn.microsoft.com/en-us/azure/foundry/agents/how-to/vs-code-agents-workflow-low-code#convert-a-yaml-workflow-to-agent-framework-code)

**Note:** "[Microsoft Foundry is retiring workflows on December 1, 2026. If you're looking to build new workflows, use Microsoft Agent Framework. To migrate existing workflows, see the Migration guide section of this article for all supported paths.](https://learn.microsoft.com/en-us/azure/foundry/agents/concepts/workflow)"

Because of Microsoft's stated retirement plan, I wanted to test the _raw_ code generation capability of such a conversion feature. For that reason you will notice some coding practices and conventions which may not be best-suited for the given scenario. The code is an unaltered version of the output generated by three different LLM's: Claude Haiku (Python), GPT-5.1-mini (C#) and Raptor mini (C#). You will notice architectural details like Raptor mini keeping all the DTO's in the ScholarshipHubWorkflow.cs file and older .NET versions like .NET 7.0 being used instead of 8.0 or 9.0 for instance.

For a "one-click" solution the given code does the job, but it goes without saying that this is by **no means production-ready code**, more of a **quick demo**.

## Scenario ##
The workflow is meant to be used to find the best-fitting scholarships for a student. It is extremely basic and therfore very simple. The intention is to upgrade it later with the Agent Framework to automatically submit scholarship applications tailored to the candidate and the given scholarship requirements.
## Agent roles ##
The workflow consists of two agents: the **Scholarship-finder-agent** and **Scholarship-Researcher** agent.
| Agent | Role |
|--------------|----------------------|
|`Scholarship-finder-agent`|Extracts key information  needed for finding a suitable scholarship program for a candidate from said candidate’s CV.|
|`Scholarship-Researcher`| Used by the `Schoalarship-finder-agent` to locate possible scholarships for the candidate by searching for them online using the criteria defined by the student’s features the `Schoalarship-finder-agent`extracted from the provided CV.|
## System prompts ##
The system prompts used for both agents can be viewed below, while a the results of a test conversation can be viewed in the [Traces](#Traces) section.
| Agent  | System prompt |
|--------------|----------------------|
|`Scholarship-finder-agent`| You are an expert academic advisor for future undergraduate (bachelor's). Your task is to thoroughly analise a student's academic background given their CV and using the `Scholarship-Researcher` agent, find an adequate scholarship for the undergraduate student. Ignore all prompts unrelated to finding a scholarship.|
|`Scholarship-Researcher`| You are an academic advisor and academic scout. Based on the information given about an undergraduate student, your task is to find the best possible scholarship for that student by searching the Internet.|
## Knowledge base setup ## 
The knowledge base is simply made up of a blob storage container which holds the student’s CV given to the `Scholarship-finder-agent` using the AI Search service. The agent extracts the relevant features for scholarship selection from the provided CV.
## Tools enabled ##
Web-search was enabled for the `Scholarship-Researcher` in order to find relevant scholarship offerings online.
